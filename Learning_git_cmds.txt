1. git init - Creates a new git repository.

2. git status - Inspects the contents of the working directory and staging area.

3.git add - Adds files from the working dirctory to the staging area.

4. git diff - Shows the difference between the working directory and the staging area.

5. git commit - Permanently stores files changes from the staging area in the repository.

6. git log - Shows git commit history.

7. git mv - Use to rename the file name or move the file from one directory to another.

8. git show - Shows the previous version of our project.

9. git config - This command sets the author name & email address to be used with your commits.

10. git clone - This command is used to obtain a repository from an existing url.

11. git reset - This command unstages the file but it preserves the file content.

12. git rm - This command delets the file from working directory & stages the deletion.

13. git tag - This command is used to give tages to the specified commit.

14. git branch- This command creates a new branch.

15. git checkout - This command is used to switch from on branch to another.

16. git merge - This command merges the specified branch's history to the current branch.

17. git push - This command sends the committed changes of master branch to remote repository.

18. git pull - This command fectches and merge chamges on the remote server to working directory.

19. git stash - This command temporarily stores all the modified tracked files and list all the stashed changes.

20. git remote - This command is used to connect local repository to the remote server.

21. git fetch - This command fetch all the branches from the repository.

22. git format-patch - This command creates a patchfile, whose contents are similar to git diff but along with code it also has metadata about commits.
